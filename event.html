What are Events?

Events are actions or occurrences that happen in the browser (user interactions, page load, mouse movement, keyboard input, etc.). Event handlers let your JavaScript respond to those events and make pages interactive.

Common events: click, mouseover, keydown, change, submit, input, load, resize

1 — Click event

Triggered when a user clicks an element such as a button. Use `onclick` property or `addEventListener('click', ...)`.

// HTML:
// 
// 


const btn = document.getElementById('btnClick');
const out = document.getElementById('clickOutput');

btn.onclick = function() {
  out.textContent = 'Button clicked!';
};

2 — Mouseover / Mouseenter

`mouseover` fires when the mouse enters an element or its children. `mouseenter` is similar but does not bubble to children.

// HTML:
// 
Hover me

// 


const box = document.getElementById('hoverBox');
const hoverOut = document.getElementById('hoverOutput');

box.onmouseover = function() {
  hoverOut.textContent = 'Mouse over the div!';
};
box.onmouseout = function() {
  hoverOut.textContent = 'Mouse left the div.';
};
Hover over me
— move the mouse over the box —
Speaker notes — Mouseover

Demonstrate how `mouseover` fires when entering children as well; `mouseenter` doesn't bubble. Discuss throttling for frequent mouse events.

3 — Keydown

Fires when a key is pressed down. Use the `event` object to get `event.key` or `event.code`.

// HTML:
// 
// 


const input = document.getElementById('textInput');
const keyOut = document.getElementById('keyOutput');

input.onkeydown = function(e) {
  keyOut.textContent = 'Key pressed: ' + e.key;
};
— pressed keys will show here —
Speaker notes — Keydown

Mention differences between keydown, keypress (deprecated/less reliable), and keyup. Show how to prevent default behavior (e.g., Enter) with e.preventDefault().

4 — Change / Input

`change` fires when an input loses focus and its value changed. `input` fires on every modification (useful for live updates).

// HTML:
// 
// 


const chInput = document.getElementById('changeInput');
const changeOut = document.getElementById('changeOutput');

chInput.onchange = function() {
  changeOut.textContent = 'Value changed to: ' + chInput.value;
};
chInput.oninput = function() {
  changeOut.textContent = 'Live value: ' + chInput.value;
};
— change or input events will appear —
Speaker notes — Change / Input

Explain difference: use input for live validation or previews, and change when you want final edited value after blur or selection change.

Best practices

  • Prefer addEventListener to attach multiple handlers and for better control.
  • Remove listeners when not needed to prevent leaks (removeEventListener).
  • Debounce or throttle high-frequency events (scroll, mousemove).
  • Use event delegation when many child elements need similar handling.

Quick Reference

Attach handlers: 
el.onclick = fn;
el.addEventListener('click', fn);
el.removeEventListener('click', fn);
